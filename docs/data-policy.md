# นโยบายข้อมูลและความเป็นส่วนตัว — LUEPiN / เลขา AI

> **ฉบับร่าง (Draft)** — วันที่มีผล: `[กำหนดวันที่]`  
> ผู้ให้บริการ: `[ชื่อนิติบุคคล / ผู้ประกอบการ]`  
> ติดต่อเรื่องข้อมูลส่วนบุคคล: `[อีเมล / LINE OA สำหรับ PDPA]`

เอกสารนี้อธิบายว่าระบบ **LUEPiN (เลขา AI)** เก็บ ใช้ เปิดเผย และลบข้อมูลอย่างไร  
ให้สอดคล้องกับพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA) และข้อความที่แสดงในหน้าลงทะเบียน / หน้าเข้าสู่ระบบ

---

## 1. ขอบเขตและคำจำกัดความ

| คำศัพท์ | ความหมาย |
| --- | --- |
| **ผู้ใช้ / คุณ** | บุคคลที่ลงทะเบียนและใช้งานผ่าน LINE ID ของตนเอง |
| **ข้อมูลธุรกิจ** | Lead, Owner, Project, Task, บันทึกการติดต่อ และข้อมูลงานขายที่คุณบันทึกในระบบ |
| **ข้อมูลส่วนบุคคล** | ข้อมูลที่ระบุตัวตนบุคคลได้ เช่น ชื่อ นามสกุล เบอร์โทร ที่อยู่ LINE ID ของลูกค้า |
| **ข้อมูลที่เข้ารหัส** | ฟิลด์สำคัญถูกเก็บแบบเข้ารหัสต่อบัญชี (per-user encryption key) ก่อนบันทึกลงฐานข้อมูล |
| **ข้อมูลที่ปิดบัง (Masked / Anonymized)** | ข้อมูลที่ตัดหรือแทนที่ตัวระบุตัวตนก่อนนำไปใช้นอกบัญชีของคุณ เช่น `Lead #A12` แทนชื่อจริง |
| **Dashboard** | หน้าเว็บแอปพลิเคชันหลักสำหรับดูและจัดการข้อมูลของคุณ |
| **ทีมพัฒนา / ผู้ให้บริการ** | ผู้ดูแลระบบ LUEPiN ที่จำเป็นต้องเข้าถึงโครงสร้างระบบเท่านั้น |

## 2. หลักการสำคัญ (สรุปสั้น)

1. **ข้อมูลที่คุณบันทึกเป็นของคุณ** — เราเป็นผู้ประมวลผลและผู้ให้บริการแพลตฟอร์ม ไม่ใช่เจ้าของเนื้อหาธุรกิจที่คุณกรอก
2. **แยกตามบัญชี** — ข้อมูลถูกแยกตาม LINE ID ของคุณ ไม่ปนกับผู้ใช้อื่น
3. **ทีมพัฒนาไม่เห็นข้อมูลจริง** — ไม่เปิดดู Lead / Owner / Project ที่ถอดรหัสแล้วในกระบวนการปกติ
4. **ไม่เปิด Backend / DB ให้ผู้ใช้** — คุณจัดการข้อมูลผ่าน Dashboard และการส่งออก (Export) เท่านั้น
5. **เก็บย้อนหลัง 1 ปี** — ขณะใช้งานหรือสมาชิกที่ต่ออายุ · Grace Period หลังยกเลิก **3 เดือน** (รายละเอียดในข้อ 6)

---

## 3. ข้อมูลที่เราเก็บ

### 3.1 ข้อมูลบัญชีผู้ใช้
- LINE User ID, ชื่อที่แสดงจาก LINE (ถ้ามี)
- อาชีพ / ข้อมูลลงทะเบียนที่คุณกรอก
- สถานะแพ็กเกจ, วันหมดอายุทดลอง / สมาชิก
- การตั้งค่าเชื่อมต่อ Google (Sheet / Drive ID ที่คุณระบุเอง)
- บันทึกความยินยอม (consent)

### 3.2 ข้อมูลธุรกิจที่คุณบันทึก
- ข้อมูล Lead, Owner, Project, Task, Pipeline, บันทึกการติดต่อ
- ข้อมูลที่อาจเป็นข้อมูลส่วนบุคคลของลูกค้าของคุณ (เช่น เบอร์โทร ที่อยู่) — **คุณเป็น Data Controller ต่อข้อมูลลูกค้าของคุณ** เราเป็น Data Processor ตามคำสั่งของคุณในการใช้ระบบ

### 3.3 ข้อมูลทางเทคนิค
- Log การเข้าใช้งาน, ข้อผิดพลาดระบบ (ไม่รวมเนื้อหาธุรกิจที่ถอดรหัส)
- Metadata การใช้งาน LINE Webhook (เช่น event type, timestamp)

### 3.4 ข้อมูลที่เราไม่เก็บโดยเจตนา
- รหัสผ่าน (ระบบใช้ LINE Login)
- ข้อมูลบัตรเครดิตในฐานข้อมูลแอป (ถ้ามีการชำระเงิน จะผ่านผู้ให้บริการชำระเงินที่แยกต่างหาก)

## 4. วัตถุประสงค์การใช้ข้อมูล

| วัตถุประสงค์ | ฐานทางกฎหมาย (PDPA) | ข้อมูลที่ใช้ |
| --- | --- | --- |
| ให้บริการ Dashboard, LINE OA, ฟีเจอร์ CRM | การปฏิบัติตามสัญญา | ข้อมูลบัญชี + ข้อมูลธุรกิจ |
| เข้ารหัสและแยกข้อมูลต่อผู้ใช้ | การปฏิบัติตามสัญญา | Encryption key ต่อบัญชี |
| ส่งข้อความ/สรุปผ่าน AI (ถ้าคุณยินยอม consent ทางเลือก) | ความยินยอม | ข้อความที่ผ่านการตัดข้อมูลระบุตัวตน |
| สำรองข้อมูลไป Google ของคุณ (ถ้าตั้งค่า) | การปฏิบัติตามสัญญา / ความยินยอม | ข้อมูลตามการตั้งค่าของคุณ |
| ปรับปรุงระบบและโครงสร้าง AI | ความยินยอม / ฐานทางกฎหมายที่เกี่ยวข้อง | **เฉพาะข้อมูลสังเคราะห์หรือที่ปิดบังแล้ว** |
| แจ้งข่าวสารบริการ, การต่ออายุ | การปฏิบัติตามสัญญา / ความยินยอม | ข้อมูลติดต่อที่คุณให้ |

## 5. การเข้าถึงข้อมูล — ใครเห็นอะไร

### 5.1 คุณ (เจ้าของบัญชี)
- เห็นและแก้ไขข้อมูลธุรกิจทั้งหมดของตนเองผ่าน **Dashboard** และ **LINE**
- **ส่งออก (Export)** ข้อมูลของตนเองได้ (รูปแบบจะพัฒนาใน Dashboard — เช่น CSV / JSON)
- **ไม่มีสิทธิ์เข้าถึง** ฐานข้อมูลดิบ, Neo4j Browser, Server SSH, หรือ Backend โดยตรง

### 5.2 ผู้ใช้อื่น
- **ไม่สามารถ** เข้าถึงข้อมูลของคุณ — ระบบแยกข้อมูลตาม LINE ID และ encryption key ต่อบัญชี

### 5.3 ทีมพัฒนา / ผู้ให้บริการ

**ในกระบวนการปกติ ทีมพัฒนาไม่เห็นข้อมูลธุรกิจที่ถอดรหัสแล้ว**

| ประเภท | เห็นหรือไม่ | หมายเหตุ |
| --- | --- | --- |
| เบอร์โทร, ที่อยู่, LINE ID ลูกค้า, พิกัดแน่นอน | **ไม่เห็น** | เก็บเข้ารหัส + ไม่เปิดในเครื่องมือ dev ปกติ |
| ชื่อลูกค้า / ชื่อโครงการจริง | **ไม่เห็น** | แม้ชื่ออาจดูไม่ sensitive แต่ยังระบุตัวตนได้ — จึงไม่นำ production decrypt ไปฝึก AI |
| โครงสร้างตาราง, schema, error log | เห็นได้ | เพื่อดูแลระบบ |
| ข้อมูลสังเคราะห์ / masked สำหรับพัฒนา graph (Neo4j) | เห็นได้ | เช่น `Lead #A12` → `Project ทาวน์โฮม ชลบุรี` (ไม่มีชื่อจริง) |
| ข้อมูล production decrypt | **เฉพาะเหตุฉุกเฉิน** | ต้องมีเหตุผลทางเทคนิค, บันทึก audit log, จำกัดผู้เข้าถึง |

**การเข้าถึงฉุกเฉิน (Break-glass)**  
เมื่อจำเป็นต้องแก้ปัญหาร้ายแรง (เช่น ข้อมูลเสียหาย, ผู้ใช้ร้องขอการกู้คืนที่ยืนยันตัวตนแล้ว) ผู้ดูแลระบบที่ได้รับอนุญาตเท่านั้นที่เข้าถึงได้ ต้องมีบันทึกว่า ใคร เข้าเมื่อไหร่ เพื่ออะไร

### 5.4 ผู้ประมวลผลภายนอก (Sub-processors)

| ผู้ให้บริการ | วัตถุประสงค์ | ข้อมูลที่ส่ง |
| --- | --- | --- |
| **LINE Corporation** | Login, Messaging API | LINE ID, ข้อความตามการใช้งาน |
| **Google (Gemini API)** | ประมวลผล AI | **เฉพาะเมื่อคุณยินยอม** — ข้อความที่ผ่านการตัดข้อมูลระบุตัวตน |
| **Google Sheets / Drive** | สำรองตามที่คุณตั้งค่า | ข้อมูลที่คุณอนุญาตให้ sync ไปยังบัญชี Google ของคุณ |
| **ผู้ให้บริการโฮสติ้ง** | เก็บเซิร์ฟเวอร์และฐานข้อมูล | ข้อมูลเข้ารหัสบน storage |

รายชื่อ sub-processor อาจอัปเดตเป็นระยะ เราจะแจ้งการเปลี่ยนแปลงสำคัญผ่านช่องทางที่เหมาะสม

## 6. ระยะเวลาเก็บข้อมูล (Retention)

### 6.1 ขณะใช้งานอยู่ / สมาชิกที่ต่ออายุ
- ดูข้อมูลย้อนหลังใน Dashboard ได้ **1 ปี (12 เดือน)** นับจากวันที่บันทึก
- ข้อมูลเก่ากว่า 1 ปีอาจถูกลบออกจากระบบแสดงผลหลัก — ต้องการเก็บนานกว่านี้ ติดต่อแอดมินผ่าน LINE

### 6.2 เมื่อไม่ต่ออายุ / ยกเลิกบริการ
1. **ระบบหลักปิดการใช้งาน** — ไม่สามารถบันทึกใหม่หรือใช้ AI ผ่าน LINE ได้
2. **Grace Period 3 เดือน** — คุณยังเข้า Dashboard แบบ **อ่านอย่างเดียว** และ **ส่งออกข้อมูล** ได้
3. **หลังครบ 3 เดือน** — ข้อมูลถูกลบออกจากเซิร์ฟเวอร์ของเรา ยกเว้นที่กฎหมายกำหนดให้เก็บต่อ

### 6.3 ข้อมูลสำรองใน Google ของคุณ
- หากคุณตั้งค่า Google Sheet / Drive ข้อมูลสำเนาอยู่ในบัญชี Google ของคุณ **ไม่ถูกลบโดยเรา** เมื่อยกเลิกบริการ — คุณจัดการเองได้

### 6.4 บันทึกทางเทคนิค
- Log ระบบอาจเก็บสั้นกว่า (เช่น 30–90 วัน) เพื่อความปลอดภัยและแก้ปัญหา

---

## 7. ความยินยอม (สอดคล้องหน้าลงทะเบียน)

### 7.1 ความยินยอมที่จำเป็น (ต้องติ๊กถึงจะใช้งาน)
> ยอมรับข้อตกลงการใช้งาน และยินยอมให้ระบบ LUEPiN จัดเก็บข้อมูลส่วนบุคคล ข้อมูลดีล และข้อมูลอสังหาริมทรัพย์ที่ฉันบันทึก **เพื่อนำไปประมวลผลภายในแอปพลิเคชันส่วนตัวของฉันเท่านั้น**

ครอบคลุม: การให้บริการ CRM, Dashboard, LINE, การเข้ารหัสต่อบัญชี

### 7.2 ความยินยอมทางเลือก (ไม่บังคับ)
> ยินยอมให้ระบบส่งต่อข้อมูลข้อความที่ผ่านการตัดข้อมูลระบุตัวตน (Anonymized Data) ไปยังระบบ AI หลังบ้าน (เช่น Google Gemini API) เพื่อประมวลผล สรุป และช่วยโต้ตอบ

- ปิดได้ตลอดเวลาในการตั้งค่าบัญชี
- เมื่อปิด ฟีเจอร์ที่พึ่ง AI ภายนอกอาจหยุดทำงานบางส่วน

### 7.3 ความยินยอมสำหรับการพัฒนาระบบ (อนาคต — แยกชัดจากข้อ 7.2)
หากในอนาคตมีการใช้ข้อมูลเพื่อปรับปรุงโมเดลร่วม จะขอ **ความยินยอมแยกต่างหาก** และใช้เฉพาะข้อมูลที่ปิดบังแล้วหรือสังเคราะห์ ไม่ใช้ production decrypt โดยอัตโนมัติ

## 8. สิทธิของเจ้าของข้อมูล

ภายใต้ PDPA คุณมีสิทธิ์:
- **เข้าถึง** ข้อมูลของตนเองผ่าน Dashboard
- **แก้ไข** ข้อมูลที่กรอกเอง
- **ส่งออก** ข้อมูล (Export) — โดยเฉพาะในช่วง Grace Period
- **ลบ** ข้อมูล — ติดต่อเราเพื่อขอลบบัญชีและข้อมูล (ภายในระยะเวลาที่กฎหมายกำหนด)
- **ถอนความยินยอม** สำหรับการประมวลผลที่อาศัยความยินยอม (เช่น AI ทางเลือก)
- **ร้องเรียน** ต่อสำนักงานคณะกรรมการคุ้มครองข้อมูลส่วนบุคคล (สคส.)

**ช่องทางคำขอ:** `[อีเมล / แบบฟอร์มใน Dashboard]` — เราจะตอบภายใน **30 วัน** นับจากได้รับคำขอที่ครบถ้วน

---

## 9. มาตรการรักษาความปลอดภัย

- เข้ารหัสข้อมูลสำคัญต่อบัญชี (application-level encryption)
- แยกข้อมูลตาม LINE ID / user_id
- HTTPS สำหรับการเชื่อมต่อเว็บ
- จำกัดสิทธิ์ผู้ดูแลระบบ (least privilege)
- บันทึก audit สำหรับการเข้าถึงฉุกเฉิน
- ไม่แชร์ข้อมูลธุรกิจระหว่างผู้ใช้

*มาตรการเพิ่มเติมที่แนะนำในอนาคต:* การหมุนเวียน key, 2FA สำหรับ admin, การทดสอบเจาะระบบเป็นระยะ

---

## 10. การโอนข้อมูลไปต่างประเทศ

หากมีการใช้บริการคลาวด์หรือ API ที่เซิร์ฟเวอร์อยู่ต่างประเทศ (เช่น Google, LINE) ข้อมูลอาจถูกประมวลผลนอกประเทศไทย เราจะดำเนินการตามหลักเกณฑ์ PDPA ว่าด้วยการโอนข้อมูลไปต่างประเทศ

---

## 11. ความรับผิดชอบของคุณในฐานะผู้ใช้งาน

เมื่อคุณบันทึกข้อมูลลูกค้า (Lead / Owner) ในระบบ:
- คุณต้องมี **ฐานทางกฎหมาย** ในการเก็บข้อมูลลูกค้านั้น (เช่น ความยินยอมจากลูกค้า, การปฏิบัติตามสัญญา)
- คุณต้องแจ้งลูกค้าของคุณตามที่กฎหมายกำหนด หากจำเป็น
- อย่าบันทึกข้อมูลที่ไม่จำเป็นต่อการขาย

---

## 12. การเปลี่ยนแปลงนโยบาย

เราอาจปรับปรุงนโยบายนี้เมื่อฟีเจอร์หรือกฎหมายเปลี่ยน  
- แจ้งการเปลี่ยนแปลงสำคัญผ่าน Dashboard, LINE หรืออีเมล  
- วันที่มีผลบังคับใช้จะระบุด้านบนเอกสาร  
- การใช้งานต่อหลังแจ้งเปลี่ยนแปลง ถือว่ายอมรับในกรณีที่กฎหมายอนุญาต

## 13. ข้อความสั้นสำหรับแสดงในแอป (User-facing summary)

ใช้ใน onboarding, Settings หรือหน้า Help:

> **ข้อมูลของคุณ**  
> Lead · Stock · Project · Task และข้อมูลที่คุณบันทึก เป็นของคุณ แยกตาม LINE ID ไม่ปนกับคนอื่น  
> ทีมพัฒนาไม่เห็นข้อมูลจริง — เห็นเฉพาะข้อมูลที่ปิดบังแล้วเพื่อพัฒนาระบบ  
> ดูย้อนหลังได้ 1 ปี (สมาชิกที่ใช้งาน/ต่ออายุ) · ส่งออกได้จาก Dashboard · ไม่ต้องเข้า Backend  
> หากไม่ต่ออายุ ระบบปิด แต่คุณยังดูและส่งออกได้อีก 3 เดือน
